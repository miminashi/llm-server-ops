## 前提・目的

Phase Sb-ctx-boundary で候補 J (9 層 SSM × VMM 非同期累積) が棄却され、代わりに:

- **slope(ctx) の強 ctx 依存性** (fa=1): ctx=16k→0.010, 65k→0.400, 131k→0.650 MiB/ub
- **ctx=32k 特異な区分項** δ(ub, ctx) ≈ +0.24 MiB @ ub=1586
- **新候補 K** (FA/attention workspace の ub×ctx cross 項) を提示

本 Phase の目的は、候補 K の**強い予測**「FA を無効にすれば slope(ctx) の ctx 依存性が消え、δ 項も消失する」を数値検証すること。

### 候補 K 支持条件（3 つ、2 つ以上満たせば支持、0/3 で棄却、3/3 で完全支持）

1. **slope 縮小**: 全 ctx × 両 Δ で ≤ 0.05 MiB/ub (fa=1 ctx=16k レベル)
2. **ctx 非依存化**: 全 8 値 (4 ctx × 2 Δ) の max/min ≤ 2.0
3. **δ 項消失**: ctx=32k で |Δ(1585→1586)| ≤ 0.05 MiB

## ボトルネック・副次発見の分析

### 1. 候補 K の事実上棄却と新解釈「FA は cross 項を suppress する最適化」

候補 K の原仮説「FA workspace が ub×ctx cross 項の**発生源**」は **正反対**だった。実測は:

- fa=1 で slope(ctx=16k) = 0.010 MiB/ub（cross 項ほぼ消失）
- fa=0 で slope(ctx=16k) = 2.120 MiB/ub（cross 項が露出、200 倍大）

したがって **FA は cross 項を発生させる側ではなく、抑制する最適化**である:

- 通常の attention: `softmax(Q·K^T)·V` を full matrix (ub × ctx) で計算 → full workspace ∝ ub×ctx
- FA: 同じ計算を tiled online 方式で実施 → workspace は tile サイズ (~O(ub) or O(ctx))、cross 項がほぼ消失

fa=1 で観測された「**弱い** slope(ctx) 依存性」(0.010 → 0.650) は FA workspace の**残存量子化項**（tile 境界での離散挙動）。δ 項 (ctx=32k ub=1586) は tile 境界跨ぎの量子化 step。

### 2. ctx=32k CUDA1 OOM の機構

fa=0 × ctx=32k で CUDA1 が 6744-6752 MiB 要求し OOM。P100 の 16 GiB のうち:

- 各層のモデル weight: 数百 MiB
- KV cache: 192 MiB/GPU @ ctx=32k
- RS buffer: 37 MiB
- **compute buffer (attention workspace 含む): 要求 6744 MiB**
- Sbctx fa=1 同条件: CUDA1 compute buffer ≈ 550 MiB → fa=0 で **12 倍以上**

fa=0 の attention full matrix workspace は ub=1584 × ctx=32k × bytes (f16 で 2 bytes) × heads = O(GiB) 単位になる。fa=1 での ~550 MiB から 6700 MiB への拡大は、full matrix workspace が露出した典型。

### 3. CUDA3 の「ctx 不依存」性質は FA 特有（新規重要発見）

Phase R / Sb-alloc / Sbctx で観測された **CUDA3 の slope = 0.98 MiB/ub で ctx 完全不依存** は fa=1 のみ。fa=0 では CUDA3 も slope 2.195 と他 GPU と同等に。

前 Phase で「CUDA3 は attention に関与しない層のみ保持」と推定したが、実際は「FA によって ctx 依存成分が抑制されるため、残った ub 線形項のみが見える」だった。fa=0 では ctx 依存成分も露出する。

### 4. graph nodes +59 の由来（推定）

fa=1 → fa=0 で graph nodes が 4473 → 4532 (+59) に増加。Qwen3.5-122B-A10B の 48 層中 attention 含む層が推定 40 層以上で、各層 FA 有無で 1-2 ノード差とすれば +59 は整合的。具体的には:

- fa=1: `flash_attn_ext(Q, K, V, mask)` 1 ノードで完結
- fa=0: `Q·K^T` + `softmax` + `mask 適用` + `·V` + `reshape` 等で 4-5 ノードに分解

layer 数が 48 で 59/48 ≈ 1.23 → 一部層で +2 ノード、他は +1 ノードの混合。

### 5. 候補 I-c 単独最有力 → 新候補 L 浮上

候補 K 棄却を受けて残る仮説:

- **候補 I-c (build_graph 内の ub 依存離散処理)**: ub の離散性は説明するが slope(ctx) 依存は未説明
- **新候補 L (FA tile 境界の量子化副作用)**: fa=1 slope(ctx) は FA tiling の不完全性による残存量子化項

新候補 L の具体化:

```
fa=1 CUDA0 compute buffer = ConstFA(ctx) + LinearFA(ctx)·ub + TileQuantFA(ub, ctx)
  LinearFA(ctx):      FA tile reduction 残差、ctx=16k で ~0 MiB/ub、ctx=65k+ で 0.4-0.65
  TileQuantFA(ub, ctx): tile 境界 (ub=1586 @ ctx=32k) での量子化 step ~0.24 MiB
```

`ggml-cuda/fattn*.cu` の内部実装で、tile size が ub の関数として量子化されている箇所を dump すれば直接確認可能。

### 6. fa=0 で δ 項が存在しない可能性

ctx=16k × ub ∈ {1584, 1585, 1586} で **完全線形**（Δpre = Δstep = +2.12、peak_ub=1585 タイ）。前 Phase fa=1 では ctx=16k も peak_ub=1585 で線形だったため、ctx=16k では δ 項非観測は fa 共通。

重要な検証点は **fa=0 × ctx=32k で δ 項が出るかどうか** だが、OOM のため未検証。OT_REGEX を拡大して CPU オフロードを増やせば fa=0 ctx=32k が実現できる可能性がある（再実施候補）。

### 7. CUDA0/1/2/3 の slope 均一化（fa=0）

fa=0 では CUDA0/1/2/3 全てで slope ≈ +2.2 MiB/ub（CUDA1 は +2.195、CUDA2 は +2.185、CUDA3 は +2.195）で数値がほぼ同じ。これは attention full matrix workspace が「層を跨いで均等に配置される」ことを示唆。一方 fa=1 では CUDA0 独占で slope 0.01 (ctx=16k)、CUDA3 は 0.98（ctx 不依存）と**割当ポリシーが全く違う**。

FA on/off で tensor scheduler の割当戦略が変わっている可能性がある（graph nodes +59 と併せて要調査）。

## 確定モデル（更新版）

前 Phase Sb-ctx-boundary 時点:
- 候補 J 棄却、候補 I-c 残存、**新候補 K 最有力**

本 Phase 後:
- 候補 K **事実上棄却**（fa=0 で逆方向 200 倍増）
- **候補 I-c 単独残存**（ub 離散のみ説明）
- **新候補 L (FA tile 量子化副作用)** 提示: FA は cross 項を suppress する最適化、fa=1 slope(ctx) と δ 項は FA tile 不完全性の残差

**2 次元 CUDA0 compute buffer モデル（fa 依存版）**:

```
fa=1:  Buf(ub, ctx) ≈ Buf0_fa1(ctx) + slope_fa1(ctx)·ub + δ_fa1(ub, ctx)
  Buf0_fa1(ctx)  : 16k→980, 32k→980, 65k→1162, 131k→1558 MiB
  slope_fa1(ctx) : 16k→0.010, 65k→0.400, 131k→0.650 MiB/ub（ctx に強依存）
  δ_fa1(ub, ctx) : ctx=32k × ub≈1586 でのみ +0.24 MiB（tile 跨ぎ量子化 step）

fa=0:  Buf(ub, ctx) ≈ Buf0_fa0(ctx) + slope_fa0(ctx)·ub + δ_fa0(ub, ctx)
  Buf0_fa0(ctx=16k) ≈ 3537 MiB（実測 3890.91 − 2.12·1584 から推定）
  slope_fa0(ctx=16k) ≈ 2.12 MiB/ub（ctx=32k では CUDA1 で ~4.26 と更に大）
  δ_fa0(ub, ctx)    ≈ 0（ctx=16k で完全線形、ctx=32k+ は OOM で未測定）
```

fa=0 は fa=1 の ~200-500% の compute buffer を要求し、**ctx≥32k で P100 の CUDA1 が OOM**。fa=1 は事実上 P100 で ctx=131k まで到達可能な唯一の経路。

## 補足

### Phase Sb-fa0 の核心発見（サマリ）

1. **候補 K 事実上棄却**: FA は cross 項の**発生源ではなく suppress する最適化**。fa=0 で slope(ctx=16k) が 0.01 → 2.12 MiB/ub と 200 倍増
2. **新候補 L 浮上 (FA tile 量子化副作用)**: fa=1 で観測された slope(ctx) と δ 項は FA tiling の不完全性による残存量子化項
3. **ctx=32k CUDA1 OOM 発見**: fa=0 で compute buffer が ~6.7 GiB/GPU に達し、P100 16 GiB で収容不能
4. **CUDA3 ctx 不依存性は fa=1 特有**: fa=0 では他 GPU と同等の slope。前 Phase の「attention 非関与」推定は FA 前提
5. **graph nodes +59**: fa=1 (4473) → fa=0 (4532)。attention 層の FA 有無でノード分解が異なる
6. **全 GPU slope ≈ 2.2 MiB/ub (fa=0)**: 割当ポリシーが FA off で均一化
7. **OOM ログから CUDA1 slope の fa=0 × ctx=32k 派生抽出**: +4.26 MiB/ub、fa=1 の 17 倍
