## 前提・目的

直前 Phase S-eval-6session (n=6) は (1) ub=1586 S6 で初崩壊（15.321→14.727、-0.594、過去最ロバスト称号反転）、(2) ub=1664 S6 過去最高値（15.292、+0.579 ジャンプ）、(3) **新ピーク順序 mode C（1664>1584>1586）発見**、(4) 2 モード仮説 (A/B) 棄却、(5) ub=1584 崩壊頻度 1/6=16.7%、(6) S4 型共通低速 vs S6 型 ub 別逆方向の 2 種類異常 session パターン発見、を報告した。

その最重要 TODO 筆頭として **「Phase S-eval-7session（mode C 継続性検証、3 モード以上 or ub 別独立変動モデル特定、ub=1586 復帰判定、ub=1664 反転継続判定）」**（所要約 50 分）が残置されていた。

本 Phase は **S6 終了から 17 分後の第 7 session（S7）を追加**し、以下を達成する:

1. **mode C 順序の S7 再現性**（6 session 中 1 回のみ → 単発 or 継続）
2. **ub=1586 の S7 復帰 vs 連鎖崩壊の判別**
3. **ub=1664 の S7 持続上昇 vs 反転下落の判別**
4. **ub=1584 崩壊頻度 n=7 更新** + **ub=1586 崩壊頻度 n=7 計測**（Wilson 95% CI 絞り込み）
5. **3 モード以上 or ub 別独立変動モデルの判定**
6. **pooled 35-run mean / σ**
7. **warmup1 ub=1584 absolute 帯のモード判定**（mode_C +0.017 帯への該当確認）
8. **σ_pool/σ_run_avg 倍率の継続観測**

### 判定しきい値

- **partial_drift**: range ≤ 0.10 t/s
- **session_dominated**: range > 0.10 t/s

## 再現性分析と解釈

### 1. mode C（S6 の 1664>1584>1586）は単発の外れ値

S7 でピーク順序は **(1586, 1584, 1664) = mode B 順序**に復帰。S6 mode C は 7 session 中 1 回のみで再出現せず、Wilson 95% CI [0.4%, 53.5%] と幅広だが、本質的には「ピーク順序が 6 session 中ほぼ常に mode A or mode B」という従来観察に回帰。S6 mode C は **非定常な外れ session** と考えるのが妥当。

### 2. ub=1586 崩壊の bouncing 機構が確定

S5 +0.187 上振れ → S6 -0.594 大崩壊 → S7 +0.407 大復帰。ub=1586 の S6 崩壊は **単発・自己回復型** で、連鎖崩壊や恒久シフトは観測されず。bouncing oscillation の振幅は ub=1586 では ±0.6 程度。

### 3. ub=1664 の長期上昇仮説は完全棄却

S5→S6 +0.579 過去最高ジャンプ（S6 報告で「長期上昇傾向復活」と評価）は S6→S7 -0.556 でほぼ完全に打ち消された。ub=1664 は **散発的 outlier 型**（多くの session で 14.59-14.74、稀に > 15.0 にスパイク）。S6 はそのスパイク session の 1 例に過ぎず、上昇トレンドの兆候ではなかった。

### 4. ub 別独立変動モデルが決定的支持

S7 で 3 ub の最近接モードがすべて異なる（1584→C、1586→A、1664→B）。これは「session 全体が 1 つの mode に支配される」モデルでは説明不能で、**ub 別に独立した状態変動が並行して起きている**ことを意味する。S1-S6 で「mode A→B→C」と思われた現象は、実は **ub 別ノイズの偶然の同方向シフト** に過ぎなかった可能性が極めて高い。

帰結:
- 「mode 数」概念自体が ub 横断では成立しない
- 各 ub 独立に「ベース帯 + 散発 jitter」のモデル化が妥当
- ub=1664 は「baseline 14.6-14.7 + 稀な +0.5 スパイク（17%確率）」
- ub=1586 は「baseline 15.13-15.32 + 稀な -0.4 ディップ（14%確率）」
- ub=1584 は「baseline 15.20-15.47 + 稀な -0.6 ディップ（14%確率）」

### 5. ub=1664 の「崩壊」定義は再考が必要

ub=1664 は 7 session 中 4 session で eval_mean < 15.0。「崩壊」と呼ぶより **「< 15.0 が baseline、> 15.0 が稀な上振れ」** と再定義すべき。

ub=1584 / ub=1586 は逆に「> 15.0 が baseline、< 15.0 が稀な下振れ」。

### 6. within-session σ は史上最低クラス、between-session σ は別次元

S7 の within-session σ:
- ub=1584: 0.004 (相対 0.026%)
- ub=1586: 0.003 (相対 0.020%)
- ub=1664: 0.002 (相対 0.014%)

これは 7 session 中の最小値クラス。一方 σ_pool/σ_run_avg = 67.2x / 42.7x / 83.5x は **session 内ノイズと session 間 drift の規模が桁違い** であることを再確認。**短時間（数分）スケールでは極めて再現性が高いが、十数分のセッション境界をまたぐと大ドリフト**という性質が完全固定。

### 7. warmup1 ub=1584 absolute 帯に第 4 帯出現

S7 の warmup1 ub=1584 = 15.418 は:
- mode_A 帯 (S1-S3: 15.51-15.78): **該当せず**
- mode_B 帯 (S4-S5: 14.78-15.37): **該当せず**
- 単独帯 15.418

Δ=+0.296 は mode_A_delta (+0.30〜+0.31) に最近接で判定では mode_A_delta となるが、absolute 帯は **第 4 帯**。warmup1 absolute / Δ の組み合わせ帯を mode の決定的識別子にすることは不適切で、**warmup1 値もセッション独立に変動する** ことを再確認。

### 8. cool time 17 分で大ドリフト（S6→S7）

S5-S6 16 分、S6-S7 17 分で連続して 16-17 分 cool time。これはこれまでの 14-21 分レンジ内で平均的な間隔。にもかかわらず ub=1586 +0.407、ub=1664 -0.556 の大ドリフト発生。**cool time と drift 量に相関なし** という前 Phase 結論を再確認。

### 9. 計測プロトコル自体の確度

7 session 連続で:
- 起動失敗 0 件
- run 失敗 0 件
- predicted_n=256 完走率 100%
- compute buffer MiB 一致率 100%

計測プロトコル自体は完全に頑健。ドリフトは GPU/host 状態起源と確定。

## 補足

### Phase S-eval-7session の核心発見（サマリ）

1. **mode C（S6）は単発**: S7 でピーク順序 (1586, 1584, 1664) = mode B 順序復帰、mode C は 7 session 中 1 回のみで再現せず
2. **ub=1586 完全復帰**: S5 +0.187 → S6 -0.594 → S7 +0.407 の bouncing oscillation、bouncing 機構確定
3. **ub=1664 反転下落**: S6 +0.579 過去最高ジャンプを S7 -0.556 で完全打ち消し、長期上昇仮説完全棄却、**散発的 outlier 型** と確定
4. **ub 別独立変動モデルが決定的**: S7 で 3 ub の最近接モードが **すべて異なる**（1584→C / 1586→A / 1664→B）、「session mode」概念は ub 横断では成立せず
5. **ピーク順序 7-session 集計**: mode A 3/7（S1-S3）、mode B 3/7（S4, S5, S7）、mode C 1/7（S6）、1 位 ub は ub=1584 と ub=1586 が同率 3/7
6. **崩壊頻度 7-session 確定**: ub=1584 1/7=14.3%、ub=1586 1/7=14.3%、**ub=1664 4/7=57.1%（過半数崩壊）**
7. **ub=1664 は崩壊定義の見直しが必要**: < 15.0 が baseline、> 15.0 が稀な上振れ
8. **compute buffer 7 session 完全一致**: ub=1586 CUDA0=980.36 / CUDA1=452.31 / CUDA2=452.31 / CUDA3=1558.12 / Host=235.48 MiB
9. **warmup1 absolute 第 4 帯出現**: S7 で 15.418、過去 mode_A (15.51-15.78) / mode_B (14.78-15.37) いずれにも非該当
10. **pooled 35-run**: ub=1584 **15.184 ± 0.253** / ub=1586 15.122 ± 0.175 / ub=1664 14.880 ± 0.256、3 ub すべて σ_pool が n=30→35 で初の縮小（bouncing 効果）
