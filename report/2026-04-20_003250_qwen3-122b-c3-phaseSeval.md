## 前提・目的

直前 Phase Sb-fa0-offload で **候補 L (FA tile 量子化副作用) の support が確定**（δ_fa0(ctx=32k) = +0.01 MiB ≈ 0 vs fa=1 δ = +0.24 MiB）。compute buffer 側の ub 境界構造は fa=1 固有の FA tile 現象として数値的に裏付いた。一方、eval 性能側で Phase Sb-fine 系が 1-run 単発で観測した以下 3 ピーク値の **Run 間再現性は未検証**だった。

|---|---|---|---|---|
| Sbfine3 | **1586** | 1k | **15.466 t/s** | Phase Sbfine3 報告値 |
| Sb-fine | 1664 | 1k | 15.451 t/s | Phase Sb-fine 報告値 |
| Sbfine2 | 1584 | 1k | 15.293 t/s | Phase Sbfine2 報告値 |

本 Phase は直前レポート未検証事項の **★最優先「ub=1586 eval 15.466 t/s の 5-10 run 再現性」** （および Sbfine/Sbfine2 継続の 1664/1584）に対応。3 条件 × (warmup 2 + eval 5) run の系統的計測で σ 付き平均を取得し、1-run 参照値との再現性を verdict 判定する。

- **confirmed**: |5-run mean − 1-run 参照値| ≤ 0.05 t/s
- **partial**: 差 ≤ 0.10 t/s
- **reject**: 差 > 0.10 t/s

## 再現性分析と解釈

### 1. run 間ゆらぎ（σ）は極めて小さい

3 条件いずれも σ = 0.005-0.008 t/s、相対誤差 0.03-0.05%。一方で **5-run mean と 1-run 参照値の乖離は −0.087 〜 −0.805 t/s** と σ の 18-160 倍。**本 Phase 内の run 間ゆらぎでは説明できない**。乖離はセッション間ゆらぎまたは 1-run 値の外れ値性に起因。

### 2. 1-run 参照値はセッション間ゆらぎを含む外れ値だった蓋然性が高い

過去 Phase（Sbfine / Sbfine2 / Sbfine3）はそれぞれ別セッション・別日で 1-run ずつ測定された値。本 Phase は **同一セッション内で連続測定**。3 つの過去 1-run 値がすべて本 Phase 平均より高い（ub=1586: +0.278, ub=1664: +0.805, ub=1584: +0.087）方向にバイアスしているのは、以下いずれかのメカニズムが考えられる:

- **過去測定日のシステム状態**（DRAM ページ配置、kernel cache、thermal 状態、他プロセス負荷）が今回より "最適"
- **Phase 間で報告された 1-run 値は、複数試行のうち最高値のみを記録**していた可能性（過去レポートの記述を改めて検証要）
- llama.cpp build 差: 同一 build バイナリを使用と想定していたが、中間で upstream 取り込み等があったか要確認

現時点で確定できるのは **5-run mean (σ=0.005-0.008) が最も信頼できる性能値** であり、**1-run 参照値は単独セッション限定の上振れ**ということ。

### 3. ピーク ub 構造の変化

1-run 時の "ub=1586 がピーク" という Phase Sbfine3 の主要結果は本 Phase で再現されず、代わりに **ub=1584 > ub=1586 > ub=1664** の順序が確立。ただし 1584 vs 1586 差は −0.019 t/s と極小で、別セッションで容易に逆転し得る。

候補 L (FA tile 量子化副作用) による compute buffer の +0.24 MiB/ub 境界ジャンプ自体は Phase Sb-fa0-offload で数値確定済み（buffer 側の fa=1 固有現象）。一方で eval 性能への影響は **ub=1586 が ub=1584 より −0.019 t/s 低い = ほぼ無影響 or わずかに悪化**。これは「tile 量子化ジャンプは buffer サイズの非連続点だが、tile 内計算効率が最適なのは境界直前の ub=1584 側」と解釈できる。

### 4. ub=1664 の大幅悪化 (−0.56 t/s) は FA tile 内の別機構

ub=1664 は ub=1584/1586 より **0.55-0.56 t/s (約 3.7%) 低い**。これは FA tile 境界 (ub=1585→1586 の +0.24 MiB ジャンプ) とは別スケールの現象。仮説:

- ub=1664 で tile 数が増加（FA parallel_blocks が 1584 付近の値と異なる tile 分割に切替わる）
- tile 数増による kernel launch overhead
- または ub=1664 が別の 2 次境界 (ub=1792/1664/1536) を跨ぐ

### 5. warmup 効果の ub 依存

ub=1584 のみ warmup 1 run で 15.510 t/s と eval mean より +0.304 t/s 高い値。ub=1586/1664 では warmup 1 ≒ warmup 2 ≒ eval mean。ub=1584 で初回効果が現れる理由は不明、kernel cache の状態に依存する現象か。ただし eval 本走 run1 は 15.208 で warmup 1 の 15.510 とは差 0.302 t/s、これは eval prompt 処理に入ってからは別 kernel が走るためと推察。

### 6. 計測プロトコル自体の確度

- run 間 cooldown 60 秒 + uniq prefix 付き prompt → prompt cache hit は発生せず（prompt_tps 68 t/s で毎 run 計算実施を確認）
- predicted_n=256 全 run 完走
- σ=0.005-0.008 は本計測手法の精度限界（ほぼ時刻測定誤差のみ）
- 本 Phase のプロトコル自体は信頼できる
