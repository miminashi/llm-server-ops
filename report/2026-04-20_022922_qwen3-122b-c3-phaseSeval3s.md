## 前提・目的

直前 Phase S-eval-cross-session は 2 session のみの cross-session 比較で、ub=1586 のみ session_independent（Δsession=+0.016 ≤ 0.02）、ub=1584/1664 は session_dominated と判定した。しかし **n=2 では σ_session が外れ値に過度に影響される** ため、直前レポートの ★最優先 TODO「**3+ session での cross-session 検証**」および次推奨 Phase 筆頭「**Phase S-eval-3session**」として残置された。

本 Phase は **時間を空けた第 3 session（S3）を追加**し、以下を達成する:

1. σ_session（セッション間標準偏差）の確度向上（n=3 で初の 3 点推定）
2. ub=1586 の session_independent 性が第 3 点で維持されるか検証
3. ub=1584/1664 の 2 セッション分布が bimodal か trimodal かを確認
4. pooled 15-run mean / σ の算出（「真の性能値」推定確度の更新）
5. ピーク ub 順序のセッション間安定性を 3 session で再確認

### 判定しきい値

- **fully_independent**: 3-session range (max−min) ≤ 0.02 t/s
- **partial_drift**: range ≤ 0.10 t/s
- **session_dominated**: range > 0.10 t/s

## 再現性分析と解釈

### 1. ub=1586 の session_independent 主張は n=2 では時期尚早だった

前 Phase S-eval-cross-session は 2 session のみで ub=1586 Δ=+0.016 を観測し「session 独立の基準点」と結論した。本 Phase S3 で −0.050 の下振れが加わり、**3-session range=0.058 > TH_INDEP (0.02)** となり verdict は partial_drift へ降格。

- **教訓**: **n=2 では "independent" 判定は外れ値 1 点のリスクが大きい**。少なくとも 3 session が最低必要、可能なら 5+ が理想。
- **ub=1586 の pooled σ は依然最小 (0.026)**: 他 ub (0.112, 0.219) の 1/4 〜 1/8、**「最もロバスト」だが「完全独立」ではない**。

### 2. ub=1664 の session 単調増加パターン

- S1=14.646 → S2=15.042 (+0.396) → S3=15.135 (+0.093)
- 増分は S1→S2 > S2→S3 で減衰、**漸近的な上昇** の可能性

**候補機構**:

- (a) **GPU thermal accumulation**: 短時間に 3 session 走らせたことで GPU / CPU が warm になり性能向上（cold 状態から漸近）
- (b) **DRAM page layout の学習的最適化**: kernel の NUMA migration / page cache の蓄積
- (c) **llama.cpp internal allocator の LRU cache 的挙動**: buffer reuse 効率化

本 Phase 時点では識別不能。**cold-boot 直後での計測が必要**。

### 3. ub=1584 のバタつき

S1=15.206 (低) → S2=15.470 (高) → S3=15.359 (中) で **trimodal に近い散布**。ub=1584 warmup 1 の +0.3 上振れが 3 session 連続再現したことから、**ub=1584 では「初回 kernel 状態」が性能を強く支配**、session 開始後の定常状態への収束までに数 run 必要と推定。

- S2 で eval 本走全体が高い（+0.264 vs S1）のは、session 全体が warm 気味に始まった結果
- S3 で中間値なのは、S2 直後の warm 状態が部分的に残存した結果かも（S2-S3 間 20 分）

### 4. 3 session でも「真の mean」は推定確度が不十分

pooled 15-run σ（ub=1586: 0.026, ub=1664: 0.219, ub=1584: 0.112）は **ub=1664 で run 内 σ の 50 倍超**、**ub=1586 でも 5.8 倍**。95% 信頼区間を ±2σ/√n とすると:

- ub=1586: 95% CI = ±0.013 t/s（±0.09%）→ 実用的に十分な精度
- ub=1584: 95% CI = ±0.058 t/s（±0.38%）→ 追加 session 推奨
- ub=1664: 95% CI = ±0.113 t/s（±0.76%）→ **5+ session 必須**

### 5. ピーク順序 1584 > 1586 > 1664 は 3 session で維持、ただし 1586-1664 差は接近

- S1: 1586-1664 = +0.542
- S2: 1586-1664 = +0.162
- **S3: 1586-1664 = +0.011**（有意だが接近）

**ub=1664 の上昇トレンド継続なら、次 session で 1586 < 1664 になる可能性**。ピーク ub の不変性は条件付き、現時点で S3 時点では ub=1586 がわずかに上位だが、マージンは極薄。

### 6. warmup ub=1584 +0.3 上振れ現象は 3 session で堅牢に再現

| Session | warmup1 | eval_mean | Δ |
|---|---|---|---|
| S1 | 15.510 | 15.206 | **+0.304** |
| S2 | 15.776 | 15.470 | +0.306 |
| S3 | 15.668 | 15.359 | +0.309 |

**Δ が 3 session で ±0.003 以内に収束**（極めて再現性が高い）。一方 absolute 値は session mean 依存で session ごとに異なる。**「ub=1584 warmup 1 は eval mean の +0.31 ± 0.003 t/s」という普遍則** が成立。

### 7. 計測プロトコル自体の確度

- σ_run = 0.004 t/s（本 Phase） = S2 (0.002-0.004) 同等、S1 (0.005-0.008) よりやや安定
- 起動 compute buffer が MiB 単位で一致 = 物理構成再現
- predicted_n=256 全 run 完走
- プロトコル自体は **3 session で頑健性 confirmed**

| 3-session range / σ_session | ✅ ub 別に 0.058 / 0.264 / 0.489 確定 |
| Welch t (prior vs S3) | ✅ ub=1586/1664 significant、ub=1584 not_sig |
| ピーク順序安定性 | ✅ 3 session で ub=1584 > 1586 > 1664 維持 |
| 1-run ref 再現性 | △ ub=1584 partial へ改善、ub=1586/1664 reject 継続 |
| compute buffer 再現 | ✅ 3 session で 1 MiB 単位一致 |
| GPU ロック | ✅ 取得・解放正常 |

**結論**:

1. **ub=1586 session_independent 主張は破綻**、partial_drift (range=0.058) へ訂正。ただし pooled σ=0.026 は引き続き最小で「最もロバスト」の性質は維持
2. **ub=1664 は 3 session 単調増加**（14.646 → 15.042 → 15.135）、片側ドリフト兆候
3. **ub=1584 は trimodal に近く**、ub=1586/1664 と異なる分布挙動
4. **ピーク順序 1584 > 1586 > 1664 は 3 session で完全維持**（最強の性質）、ただし 1586-1664 差は +0.011 まで接近
5. **compute buffer は 3 session すべて MiB 単位一致** → 性能ドリフトは runtime kernel 状態由来（物理構成ではない）
6. **ub=1584 warmup 1 の +0.31 t/s 上振れは 3 session で ±0.003 以内に再現**（ub=1584 固有の普遍則として確立）
7. **pooled 15-run mean は n=3 でも mean/σ が大きく動き得る**、5+ session 必須（特に ub=1664）
