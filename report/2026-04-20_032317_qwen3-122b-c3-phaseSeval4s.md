## 前提・目的

直前 Phase S-eval-3session (n=3) は ub=1586 の "session_independent" 主張を partial_drift へ降格させ、ub=1664 の「3 session 単調増加」と「ピーク順序 1586-1664 差が +0.011 まで接近」を報告した。その最重要 TODO 筆頭として **「n=3 でも verdict が覆ったため、n=4 で σ_session の収束性を確認」**（Phase S-eval-4session 候補、所要約 50 分）が残置されていた。

本 Phase は **時間を空けた第 4 session（S4）を追加**し、以下を達成する:

1. σ_session（セッション間標準偏差）の n=4 推定
2. ub=1586 partial_drift 判定が第 4 点で維持されるか検証
3. ub=1664 単調増加 (14.646 → 15.042 → 15.135) が漸近・反転するか検証
4. ピーク順序 1584 > 1586 > 1664 の 4 session 安定性確認（S3 差 +0.011 からの逆転可能性）
5. pooled 20-run mean / σ の算出（「真の性能値」推定確度の更新）
6. compute buffer の 4 session MiB 単位一致の再確認

### 判定しきい値

- **fully_independent**: 4-session range (max−min) ≤ 0.02 t/s
- **partial_drift**: range ≤ 0.10 t/s
- **session_dominated**: range > 0.10 t/s

## 再現性分析と解釈

### 1. ub=1584 が S4 で大崩壊した理由

S1=15.206, S2=15.470, S3=15.359 と「高め気味」の 3 session 分布が続いた後、S4 で 14.631 と −0.728 の大幅下振れ。**4-session range 0.839 で 3 ub 中最大**。可能な機構:

- **(a) thermal cycle**: S3 までで GPU warm に蓄積、S3-S4 の 14 分間隔で cool 方向に変化した可能性
- **(b) DRAM page migration**: session 間で NUMA/page cache 状態が再配置、S4 で ub=1584 固有の kernel path に不利な layout となった
- **(c) allocator LRU cache**: ub=1584 の特定 tensor の配置が S4 で cache miss 化

ub=1584 warmup 1 の Δ=+0.153（過去 Δ=+0.31 の 49%）も同時に崩壊。**ub=1584 固有の性能は「warmup 1 で +0.31」と一貫していたが、S4 では +0.153 に縮小、かつ eval mean も大幅低下。ub=1584 の性能は session kernel 状態に強く依存**、S1-S3 は有利条件だった可能性がある。

### 2. ub=1664 単調増加の破綻

S1=14.646 → S2=15.042 → S3=15.135 の単調上昇から、S4 で 14.593 と **S1 より低い値まで戻った**。前 Phase の「漸近的上昇（候補 a/b/c）」仮説は棄却、**ドリフトは単調ではなく bimodal または periodic な可能性**。候補機構:

- session 間で「高速状態」と「低速状態」を確率的に切り替え、S1/S4 = 低速状態、S2/S3 = 高速状態
- thermal saturation が 30 分程度のサイクルで on/off し、測定タイミング依存

### 3. ピーク順序の劇的再編成

| session | 1位 | 2位 | 3位 | 1位 − 2位 |
|---|---|---|---|---|
| S1 | 1584 | 1586 | 1664 | +0.018 |
| S2 | 1584 | 1586 | 1664 | +0.266 |
| S3 | 1584 | 1586 | 1664 | +0.213 |
| **S4** | **1586** | **1584** | 1664 | **+0.503** |

**S4 では ub=1586 が圧倒的 1 位**（2 位との差 0.5 t/s）、S1-S3 では常に 2 位だったものが S4 で 1 位に。これは **ub=1586 が「他 ub ほど崩れない」性質**が極端に表れた結果。ub=1586 自体は S4 で S3 から −0.012 と微減だが、ub=1584 が −0.728、ub=1664 が −0.542 と大きく下振れたため相対的にトップに浮上。

**この結果は ub=1586 の「最もロバスト」性質をさらに強化**: 他 ub が低速状態に落ちた時でも ub=1586 は相対的に守られる。

### 4. warmup ub=1584 +0.31 t/s 普遍則の条件付き性

3 session (S1-S3) で ±0.003 以内に収束していた Δ が S4 で +0.153 に縮小。「eval_mean + 0.31」の相加関数形ではなく、**おそらく「特定の warmup 状態では絶対 15.67 付近の kernel path に入る」という絶対値基準の現象**。eval_mean が S4 で 14.631 に落ちた結果、warmup 1 も 14.784 に追随した（差 0.153）。

- S1 eval=15.206, warmup1=15.510（差 +0.304）
- S2 eval=15.470, warmup1=15.776（差 +0.306）
- S3 eval=15.359, warmup1=15.668（差 +0.309）
- **S4 eval=14.631, warmup1=14.784（差 +0.153）** ← 破綻

warmup 1 の絶対値 15.510-15.776 → 14.784 への縮退は、warmup 1 の状態自体も session kernel 依存であることを示唆。**「Δ=+0.31 普遍則」は S1-S3 の偶然の一致だった可能性が高く**、warmup 1 も eval 同様に session 分布を持つ。

### 5. pooled 20-run mean の収束性

| ub | n=10 (S1+S2) | n=15 (S1+S2+S3) | n=20 (S1+S2+S3+S4) | Δ_(n=15→n=20) |
|---|---|---|---|---|
| 1584 | 15.338 | 15.345 | 15.167 | **−0.178** |
| 1586 | 15.196 | 15.179 | 15.168 | −0.011 |
| 1664 | 14.844 | 14.941 | 14.854 | **−0.087** |

**ub=1584 は n=20 で mean が −0.178 大幅下振れ**、前 Phase 推定値 15.345 から遠ざかった。**ub=1586 は −0.011 で安定**（4-session で mean がほぼ同一点に収束）。**ub=1664 は −0.087 で逆方向**（S3 で上昇傾向だった）。

**ub=1586 以外は pooled mean がまだ収束しておらず**、真の mean 推定には **5+ session 必須**（特に ub=1584 / ub=1664）。

### 6. S4 は session group として「低速側」

Welch t で全 3 ub prior pool 対比 significant 下振れ。S4 全体が「低速 session」の典型例。原因候補:

- (a) S3 から 14 分で S4 開始、他 session より直前間隔が短く cool time が不足
- (b) GPU サーバ側の他プロセスが S4 タイミングで活動（dmon 監視未詳細）
- (c) llama.cpp server 起動順序の何らかの性能差

本 Phase 時点では切り分け不能。**cold-boot 計測 / 翌日計測** で切り分け必要。

### 7. 計測プロトコル自体の確度

- σ_run = 0.002-0.007 t/s（本 Phase）= S1-S3 同等
- 起動 compute buffer が 4 session で MiB 単位一致 = 物理構成完全再現
- predicted_n=256 全 run 完走
- プロトコル自体は **4 session で頑健性確認**

## 採用判定

| 項目 | 結果 |
|---|---|
| 起動成功率 | ✅ 3/3 (ub=1584/1586/1664) |
| eval 5-run 取得 | ✅ 3/3 × 5 run |
| σ 取得 | ✅ 0.002-0.007 t/s（前 Phase 同等） |
| 4-session range / σ_session | ✅ ub 別に 0.839 / 0.070 / 0.542 確定 |
| Welch t (prior vs S4) | ✅ 全 ub significant（全て下振れ） |
| ピーク順序安定性 | ❌ 崩壊（S4 で ub=1586 が 1 位）、3 session 維持から変化 |
| 1-run ref 再現性 | ❌ 3 ub すべて reject（S4 全体が低速側） |
| compute buffer 再現 | ✅ 4 session で 1 MiB 単位一致 |
| GPU ロック | ✅ 取得・解放正常 |

**結論**:

1. **ub=1584 が S4 で −0.728 大崩壊**、4-session range 0.839 で最大セッション間ゆらぎ（約 5.5%）
2. **ub=1664 単調増加仮説は破綻**、S4 で S1 以下まで戻り bimodal/periodic の可能性
3. **ub=1586 は 4 session で唯一 partial_drift を維持**、range 0.070、σ=0.030 で最もロバスト
4. **ピーク順序 1584 > 1586 > 1664 は S4 で崩壊**、新ピークは ub=1586（S3 で予告された「1586-1664 逆転」ではなく「1584 崩壊」で発生）
5. **compute buffer は 4 session すべて MiB 単位一致** → 性能ドリフトは runtime kernel 状態由来
6. **warmup ub=1584 +0.31 t/s 普遍則は破綻**、Δ は session kernel に依存
7. **pooled 20-run: ub=1584 15.167 ± 0.331 / ub=1586 15.168 ± 0.030 / ub=1664 14.854 ± 0.244**
8. **ub=1584 と ub=1586 の pooled mean がほぼ同値**（差 0.001）、σ は 11 倍違い → **実効性能は ub=1586 が最高**（安定性込みで評価）
9. S4 は全 ub prior pool 対比 significant 下振れ → **session group として低速側**、thermal / inter-session interval / 他プロセス 要因 候補

### 最新のベストエスティメート（pooled 20-run mean）

- **ub=1584: 15.167 ± 0.331 t/s**（前 Phase 推定 15.345 から −0.178、σ 約 3 倍、依然 5+ session 必須）

## 補足

### Phase S-eval-4session の核心発見（サマリ）

1. **ub=1584 が S4 で 14.631 へ大崩壊（−0.728 t/s, 4-session range 0.839）**: 3 session の「高め気味」分布から一転
2. **ub=1664 単調増加仮説は破綻**: S1=14.646, S2=15.042, S3=15.135 → **S4=14.593** で S1 以下まで戻る → bimodal/periodic
3. **ub=1586 は 4 session で唯一 partial_drift 維持 (range 0.070, σ_session 0.033)**、他 ub の 1/8〜1/11 の安定性
4. **ピーク順序が S4 で崩壊**: S1-S3 は ub=1584 > 1586 > 1664、**S4 は ub=1586 > 1584 > 1664**（差 +0.503）
5. **S4 は session group として全 ub 下振れ**: prior pool 対比 Welch t で 3 ub すべて significant 下振れ、共通要因あり
6. **warmup ub=1584 +0.31 普遍則は S4 で破綻** (Δ=+0.153)、絶対値基準の現象だった可能性
7. **compute buffer は 4 session すべて MiB 単位完全一致** → 性能ドリフトは runtime kernel 状態由来
8. **pooled 20-run**: ub=1584 15.167 ± 0.331 / ub=1586 **15.168 ± 0.030** / ub=1664 14.854 ± 0.244
9. **ub=1584 と ub=1586 の pooled mean がほぼ同値**（差 0.001）、σ は 11 倍違い → **実効性能は ub=1586 が最高**
10. prompt-path ドリフトは eval-path と近値化（完全独立ではない可能性）
