## ボトルネック・副次発見の分析

### 1. CUDA3 の純 `ub` 支配は **構造的な事実** — Phase R で物理的に確定

Phase Q で `CUDA3 = 0.9824·ub、R²=1.00000000` が確立したが、これは「ctx=16k 固定で ub を変えたときの線形性」であった。Phase R で **ctx を 8 倍に拡張しても CUDA3 が誤差 0.04 MiB（+0.002%）しか変動しない**ことを実測し、以下を確定:

- CUDA3 の compute buffer は **embedding output / logits staging 専用**、ctx 側の KV や attention 計算には関与しない
- Phase Q レポート §1「0.9824 ≈ hidden_dim × 2 byte/token の物理解釈」は正しい
- 本番 start.sh の起動前 lint では **`predicted_cuda3 = 0.9824 × ub` が ctx に一切依存せず使える**

### 2. CUDA0/1/2/CUDA_Host は ctx 依存成分を持つ — Phase Q では分離不能

| GPU | ub 係数（Phase Q） | ctx 係数（Phase R 新発見） | 物理解釈（推定） |
|---|---:|---:|---|
| CUDA0 | 0.077 MiB/ub | **0.00987 MiB/ctx_token** | input embedding + attention scores partial |
| CUDA1 | 0.254 MiB/ub | **0.00391 MiB/ctx_token** | 中間層 attention KV staging + MoE router |
| CUDA2 | 0.254 MiB/ub | **0.00391 MiB/ctx_token** | 同上 |
| CUDA3 | 0.9824 MiB/ub | **0** | embedding output（ctx 無依存） |
| CUDA_Host | 0.086 MiB/ub | **0.00781 MiB/ctx_token** | KV cache pinning / DMA staging |

ctx 係数の合計 0.02549 MiB/token が「ctx 依存 overhead」を表し、ctx=131k で compute buffer 全体が **+2,923.87 MiB（+68.4%）** 増加する理由が解明。

### 3. Phase Q 「CUDA_Host 定数項ゼロ」結論の訂正

Phase Q レポート §4 で「Phase P 推定の CUDA_Host 定数項 100〜130 MiB は Phase Q で否定、実際は純比例 0.086·ub」と結論していたが、**これは ctx=16k 固定という制約の中だけで成立する話**だった。

Phase R で `CUDA_Host = 0.086·ub + 0.00781·Δctx` と判明し、ctx=131k では ctx 成分 896 MiB が支配的（ub 成分 176 MiB の 5 倍）。**Phase N→P→Q の「単一切片モデルで fit」路線の限界を Phase R が突破**した。

### 4. KV layer 数 12 の構造的証拠

起動ログ: `llama_kv_cache: size = 3072.00 MiB (131072 cells, 12 layers, 1/1 seqs)`

Qwen3.5-122B は総 48 層、C-D3 構成で `-ot 'blk\.([0-9]|1[0-3]|2[0-4]|3[1-9]|4[0-7])\.ffn_.*_exps\.weight=CPU'` により層 0〜13、24〜30、31〜47 の ffn_exps を CPU オフロード。残る GPU 上 KV cache を持つ層は:

- 層 14〜23: 10 層
- 層 25〜29? 確認中

ログの「12 layers」は **GPU 上で KV cache を保持する層数**。残りの 36 層分の KV は CPU 側に保持される。KV buffer 合計 3,072 MiB = 12 層 × 256 MiB/層（K=128 + V=128 MiB/層）とほぼ一致。

### 6. eval の長 ctx 劣化 — 120k で -17%

Phase O で確立した線形モデル `time_per_token ≈ 66.5μs + 0.485μs × N_context` に N_context=120,265 を代入すると:

```
t_per_token ≈ 66.5 + 0.485 × 120,265 / 1000 = 66.5 + 58.3 = 124.8 μs
→ eval ≈ 8.01 t/s（予測）
```

実測 12.82 t/s は予測の 160%、**Phase O モデルは保守的（悲観的）すぎる**。Phase O モデルは小 ctx 領域の線形フィットなので、長 ctx では attention の KV キャッシュヒット等の最適化効果が効いて実際には乖離する可能性あり。
