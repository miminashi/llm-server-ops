## 核心発見サマリ

**Phase D (15.03 t/s) / Phase S (15.39 t/s) を超える KV 型 × ub 組合せは存在しなかった。** KV cache 量子化は eval t/s 改善の手段として限界 (最良でも 15.016 t/s、Phase D の -0.1%)。

| 観点 | 結果 |
|------|------|
| 最良 eval 構成 | **KV=q8_0, ub=1586**, eval_mean = **15.016 t/s** (Phase D -0.1%, Phase S -2.4%) |
| 最良 prompt 構成 | **KV=f16, ub=1586**, prompt_mean = **68.963 t/s** |
| Phase D (15.03) 超え条件 | **なし** (q8_0 ub=1586 が最接近だが 15.016 < 15.03) |
| Phase S (15.39) 超え条件 | **なし** (全条件で 15.39 以下) |
| KV 量子化と eval 速度の関係 | **f16 < q4_0 < q4_1 ≈ q8_0** (低精度 ≠ 高速。q8_0 が +4.1% で最速) |
| KV 量子化と prompt 速度の関係 | **q4_1 < q4_0 < q8_0 < f16** (量子化で悪化方向、但し差は 1.8%以内) |
| 出力品質 (目視) | f16/q8_0/q4_0/q4_1 いずれも崩壊なし (1k prompt 要約タスクで整合的な reasoning 生成) |
| run 間 stdev | 0.003 - 0.139 t/s (極めて安定、session 内測定ノイズは無視可能) |

**重要な副産物的発見**: **q8_0 KV は f16 KV より eval で +4.1% 高速** (ub=1586 で 15.016 vs 14.425)。但し本 Phase の f16 baseline は S54 参照 (15.173) に対し -4.9% と下振れており、session-level 変動の範囲内の可能性が高い。

## 前提・目的

### 背景

qwen3-122b の eval t/s 改善履歴:
- **Phase A** (2026-04-15): expert layer 14-19 GPU 復帰で 10 → 12 t/s
- **Phase D** (2026-04-16): numactl -N1 -m1 --threads 40 で 12 → **15.03 t/s** (最大跳躍)
- **Phase S** (2026-04-19): ctx×ub 2D 細粒度探索で **15.39 t/s** (ctx=65k, ub=512)
- **Phase S-eval S1-S59** (2026-04-20 - 2026-04-22): 同一バイナリ反復測定の再現性検証。S1 で reject 決着済み、以降は同条件反復のみ

### 目的

Phase S-eval が「再現性検証」として情報飽和したため、**パラメータチューニングによる追加改善余地があるか** を直接検証する Phase T シリーズに切り替える。

**Phase T-1 スコープ**: 5 つの有望軸 (KV cache 量子化 / split-mode row / threads 中間値 / OT pattern / ビルドフラグ) のうち最優先の **KV cache 量子化** (`--cache-type-k` / `--cache-type-v`) をスイープし、eval t/s と prompt t/s 両面で Phase D/S ピーク超え条件の存在を判定する。

### 選定理由

- `f16` と `q8_0` のみ既測、`q4_0 / q4_1` 未検証
- 再ビルド不要で実行時フラグのみで切替可 → 低コストで完全検証可
- 仮説: KV 量子化で VRAM 圧縮 → compute buffer 余裕拡大 → より大きな -ub 許容 / memory traffic 削減 → eval 速度向上

### 判定基準

| 判定 | 閾値 |
|------|------|
| Phase S ピーク超え | eval_mean > 15.39 t/s |
| Phase D ピーク超え | eval_mean > 15.03 t/s |
| 量子化で改善 | f16 mean 比 +3% 以上 |
| 量子化で中立 | f16 mean ± 3% |
| 量子化で劣化 | f16 mean 比 -3% 以下 |

### 観察

1. **eval では q8_0 > q4_1 > q4_0 > f16** (ub=1586)。量子化による KV 読み出し帯域削減の効果は q8_0 で最大、それ以上細かくしても改善しない (脱量子化コストが支配的になる可能性)。
2. **prompt では f16 > q8_0 > q4_0 > q4_1** (ub=1586)。prompt 処理は attention 計算が dense、KV 脱量子化コストが明確に効く。
3. **ub=1586 と ub=1664 のクロス**: q8_0 / q4_0 は ub=1586 で顕著に速い (-0.77/-0.67 t/s at ub=1664)。f16 / q4_1 のみ ub=1664 の方がわずかに速い (+0.146/-0.054)。KV 型によって最適 ub が異なる = ub 探索は KV 型ごとに独立に再実施する必要がある可能性。

## 条件別詳細

### compute buffer / KV buffer サイズ

| KV 型 | KV total (MiB) | K per-layer | V per-layer | CUDA0 CB (ub=1586) | CUDA0 CB (ub=1664) |
|-------|---------------|------------|------------|--------------------|-------------------|
| f16 | 768.00 | 384.00 | 384.00 | 980.36 | 1002.61 |
| q8_0 | 408.00 (-47%) | 204.00 | 204.00 | 1030.19 (+5.1%) | 1054.87 |
| q4_0 | 216.00 (-72%) | 108.00 | 108.00 | 1030.19 | 1054.87 |
| q4_1 | 240.00 (-69%) | 120.00 | 120.00 | 1030.19 | 1054.87 |

- **KV 量子化で KV buffer は 47-72% 削減されるが、CUDA0 compute buffer は 50 MiB 程度増加** (脱量子化用一時バッファの確保による)。
- q4_0 / q4_1 / q8_0 の compute buffer はバイト一致 → 脱量子化用バッファは量子化 bit に依存しない固定サイズ。
- CUDA3 compute buffer は 1558.12 MiB (ub=1586) / 1634.75 MiB (ub=1664) でどの KV 型でも同値、ctx × ub に線形。

### run 間安定性

すべての条件で stdev ≤ 0.139 t/s、5 run 内の測定ノイズは Phase S-eval の session 間 σ (0.02-0.10) と同等かより低い。本 Phase の順位は統計的に信頼できる。

### 出力品質

1k prompt (計算機科学複数トピックの 3 項目要約タスク) に対する `reasoning_content` 冒頭を目視比較:

- **f16**: "A text discussing various aspects of computer science and engineering..."
- **q8_0**: "A text containing multiple paragraphs discussing various aspects of computer science, systems, and AI..."
- **q4_0**: "A text containing multiple paragraphs about various computer science topics..."
- **q4_1**: "A text discussing various aspects of computer science, computation, systems, AI/ML..."

**すべて論理構造と文脈理解が保たれており、崩壊なし。** 定量的 perplexity 評価は本 Phase のスコープ外 (後続 TODO)。

### f16 baseline 再現性チェック

- S54 (2026-04-22 07:24) f16 ub=1586 eval_mean = **15.173 t/s**
- 本 Phase f16 ub=1586 eval_mean = **14.425 t/s**
- Δ = **-0.748 t/s (-4.9%)** → ±5% 範囲内だが下限ギリギリ

**注意**: Phase S-eval 54-session 解析では ub=1586 の session 間 σ は ~0.09 t/s、range ~0.4 t/s。本 Phase の 14.425 は 54-session pool の下端に位置する可能性がある。**KV 型間の相対比較は信頼できるが、absolute 値の Phase D/S 比較には 0.7 t/s 程度の下振れ補正余地あり**。q8_0 ub=1586 の 15.016 も補正すれば 15.76 相当 (Phase S 15.39 超え) の可能性を否定できない。

## 未検証事項

以下は本 Phase のスコープ外、後続 Phase T-2 以降の候補:

| 項目 | 候補 Phase | 理由・期待 |
|------|-----------|-----------|
| **split-mode row vs layer** | Phase T-2 | 現行は `layer` (暗黙 default)。CUDA3 compute buffer の偏在 (1634 MiB、他 GPU の 3.4x) の根本要因で、`row` 未試行。layer 間 kernel launch overhead 削減の可能性 |
| **threads 中間値 (24/28/32/36)** | Phase T-3 | 20/40/80 のみ測定、Phase D で 40 採用。物理コア 40 枚の一部のみ利用 (中間値) が cache locality 改善する可能性 |
| **OT pattern 変種** | Phase T-4 | Phase A で `blk.14-19` のみ GPU 復帰で +18% 達成、現行は `blk.25-30` GPU 配置。他 expert 層範囲 (例 `blk.0-5` または `blk.42-47` など) の優位性未検証 |
| **llama.cpp ビルドフラグ** | Phase T-5 | `GGML_CUDA_FORCE_MMQ` / `GGML_CUDA_FORCE_DMMV` の P100 (CC 6.0) 最適化未検証。再ビルド必要のため最後 |
| **q5_0 / q5_1 KV 型** | 本 Phase 外 | q8_0 → q4_0 間の中間精度。q8_0 の相対優位性の根拠特定に有用 |
| **KV 非対称 (k=q8_0, v=f16 等)** | 本 Phase 外 | K と V で精度分離、attention score 計算 (K 読み出し) と value 加算 (V 読み出し) の帯域支配度分離 |
| **ctx=65536 での KV 量子化挙動** | 本 Phase 外 | Phase S ピーク (15.39) は ctx=65k × ub=512。本 Phase の ctx=32k とは compute buffer パターンが異なる |
| **-ub スイープ (1586/1664 以外)** | 本 Phase 外 | KV 型ごとに最適 ub が異なる可能性 (上記観察 3)。各 KV 型で独立の ub sweep が必要 |
| **f16 baseline の下振れ原因** | 要調査 | S54 15.173 に対し本 Phase 14.425 (-4.9%)。time-of-day / 温度 / 前条件の影響の切り分け |
