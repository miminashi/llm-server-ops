## 前提・目的

Phase L で f16 KV + flash-attn=0 の compute buffer が `O(n^1.3)` の混合オーダーであることが判明（Phase K 仮説 `O(n²)` の部分訂正）。ただし 2 点 (ctx=16384/4096) では `buffer(n) = a·n² + b·n + c` の 3 係数を分離不可。

本 Phase M では、**ctx=2048, 1024 の 2 点を追加** して 3 点厳密解で係数分離を行い、以下の内訳を特定する:

- `O(n²)` 成分（attention score matrix）
- `O(n)` 成分（中間活性 / logits / KV 操作領域）
- `O(1)` 成分（embedding / output layer 常駐領域）

### 成功条件

- [x] ctx=1024 / 2048 での fa=0 起動可否判定
- [x] `sched_reserve: CUDA* compute buffer size` の採取（GPU 別に 5 本）
- [x] GPU 別の 3 点厳密解による係数 `a, b, c` の推定
- [x] Phase L 予測値 (O(n^1.3) 外挿) との実測突き合わせ
- [x] 副次目的: ctx-size の eval 速度への影響を極小 ctx 側（16k/4k/2k/1k）で追加検証

## ボトルネック・副次発見の分析

### 1. CUDA3 の「ctx 不変」仮説の反証

Phase L の 6-3 節で「CUDA3 が fa=0 / fa=1 共に 4,024 MiB で不変、attention 担当外の定数領域」と推測したが、**Phase M で CUDA3 も純粋 O(n) でスケール（b=0.9824、c=0）**であることが判明。

- fa=0 ctx=4096: 4,024 MiB
- fa=0 ctx=2048: 2,012 MiB
- fa=0 ctx=1024: 1,006 MiB

係数 b=0.9824 MiB/token の物理解釈: fp16 で約 1 byte/token/gpu に近い値。KV cache 関連の中間 staging buffer の可能性。**CUDA3 が「定数成分」ではなく「線形成分の主担」**という理解にアップデート。

Phase L での「fa=0/1 で不変」という観察は ctx=4096 固定下でのみ成立した現象で、ctx を変えると線形変動することが今回明らかになった。

### 2. CUDA0 の定数成分 c=1,562 MiB の解釈

CUDA0 のみ大きな c（他 GPU では c≈0）を持つ。これは:

- **embedding 層 (vocab_size × hidden_dim) の常駐 compute buffer**（語彙テーブル経由の計算領域）
- **output head (logits 層) の常駐 buffer**
- その他「ctx に依存しないモデル構造領域」

のいずれかの組み合わせと推定。Phase K で観察された「ワークスペース +950 MiB の内訳」にも関連し、`llama.cpp` ソース解析への動機が更に強まる。

### 3. 線形モデルと n² モデルの領域分割

- **小 ctx (n ≤ 2048)**: 定数 c=1,562 MiB + 線形項 b·n が支配的 → **Phase L の単純ベキ則 (k=1.306) は適用できない**
- **中 ctx (2048 ≤ n ≤ 4096)**: 線形項と n² 項が同等 → k≈1.3 で近似可能
- **大 ctx (n ≥ 4096)**: n² 項が支配的 → Phase L の k=1.306 が正確、n→16k では k=1.306 が維持

**実用上の起動可否判定式**（P100 16GB 制約下）:
- fa=0 + ctx ≤ 2048: compute buffer 合計 ≤ 5 GB で余裕
- fa=0 + ctx = 4096: compute buffer 合計 ≈ 12 GB、ぎりぎり成立（Phase L 実証）
- fa=0 + ctx ≥ 8192: CUDA0 が 16GB を超える予測 → 起動不可（Phase K 実証）

### 4. eval 速度の非線形領域（ctx=1024 の −5.2%）

Phase L で「ctx-size の eval 速度への直接影響はほぼゼロ（ctx=4k vs 16k で 0.5%）」と結論したが、Phase M で **ctx=1024 の eval が ctx=4096 比 −5.2%** と明確な劣化を観測。

- ctx=1024 → 2048 → 4096 → 16384: eval = 14.29 → 14.78 → 15.07 → (Phase K fa=1 での 15.03 相当)
- 飽和曲線的な挙動で、**ctx ≥ 4096 で飽和**し、それ以下で漸減

要因推定:
- 小 ctx では attention 計算がパイプライン化されず、GPU 稼働率が下がる
- 小 ctx でも `graph nodes = 4532` / `graph splits = 136` が一定 → split オーバーヘッドが相対的に重い

Phase L の結論「ctx-size は eval 速度にほぼ影響しない」は **ctx ≥ 4096 の範囲で有効**、と範囲限定付きで修正。

### 5. 1k prompt eval の KV 埋まり度非依存

ctx=2048（KV 53% 使用） vs ctx=4096（KV 26% 使用）の 1k prompt eval 差は 1.8%（14.23 vs 14.50）。**KV 埋まり度合いではなく prompt_n 絶対値が eval 速度を決める**ことを示唆。

これは `n_kv × n_decoded` でスケールする fa=0 attention の計算量モデルと整合し、ctx-size より n_kv（実際に埋まったトークン数）が支配的パラメータとなる。

## 採用判定

| 項目 | 結果 |
|------|------|
| fa=0 + ctx=1024 起動 | ✅ 可能 |
| fa=0 + ctx=2048 起動 | ✅ 可能 |
| Phase L 仮説「compute buffer は O(n^1.3) 混合」| **大 ctx 域 (n ≥ 4k) で有効、小 ctx 域 (n ≤ 2k) では定数項 1,562 MiB が支配** |
| CUDA3 の ctx 不変仮説 | **反証**: 実際は b=0.9824 の完全線形 |
| eval 速度の ctx 非依存仮説 | **部分訂正**: ctx ≥ 4k で有効、ctx=1k で −5.2% 劣化 |
| C-D3 採用構成（q8_0, ctx=131k, fa=1）の妥当性 | **再確認**（fa=1 のメリットは大 ctx 側でさらに顕著）|

**結論**: Phase L で確立した「fa=1 は C-D3 の機能要件」は維持。Phase M の新規所見として:

1. **compute buffer の 3 係数分離に成功**: CUDA0 のみ大きな定数成分 c=1,562 MiB、他 GPU は c≈0、CUDA3 は純粋線形（b=0.9824）
2. **fa=0 は ctx=1024 / 2048 / 4096 の 3 段階で起動可能**、ただし eval は ctx が小さいほど劣化（ctx=1k で −5.2%）
3. **Phase L ベキ則 k=1.306 は大 ctx 域でのみ有効**、小 ctx では定数項支配のため単純ベキ外挿は 2.4x 過小予測
4. **CUDA3 の線形スケーリング発見**: Phase L の「定数成分」仮説を ctx 変動実験により反証

本番 `start.sh` の改変は不要。ただし追加ドキュメント化 TODO:
- 「P100 fa=0 の起動可否は ctx ≤ 4096」という既知情報に加え、「ctx ≤ 2k では eval も遅くなる」を注記
- 「compute buffer の実測モデル (CUDA0: 2.45e-4·n² − 0.68·n + 1563、CUDA3: 0.98·n) を skill に記録」

## 補足

### Phase M の核心発見

1. **CUDA3 は線形スケーリング (b=0.98 MiB/token)** — Phase L の「ctx 不変 4,024 MiB」は ctx=4096 限定の観察だった
2. **CUDA0 の定数項 c=1,562 MiB が ctx ≤ 2048 で支配的** — 小 ctx 外挿で Phase L ベキ則 k=1.306 を 2.4x 過小予測する原因
3. **fa=0 eval は ctx ≥ 4096 で飽和、ctx=1024 で −5.2% 劣化** — Phase L 副次所見「ctx 非依存」の部分訂正
4. **2 次多項式モデルは小 ctx 域で完璧、大 ctx 外挿で破綻** — ctx=16384 予測 145 GB は明らかに非物理、領域分割モデル必須

### 計算モデルの領域分割

```
compute_buffer_total(n) =
  - 小 ctx (n ≤ 2048):   5.01e-4·n² + 0.58·n + 1562.6   [3 点厳密解]
  - 大 ctx (n ≥ 4096):   12176.18 * (n/4096)^1.306       [Phase L log-log]
```

両領域は ctx=4096 で 12,176 MiB で連続（厳密解の予測 12,352 MiB と誤差 1.4%）。ctx=8192 は中間域で、どちらのモデルも過大予測（厳密解 39,953、ベキ則 30,033）の可能性あり、次 Phase で実測確認が必要。

### K_f16_fa0 対 M_f16_fa0 の数値サマリ

Phase K では fa=0 ctx=16384 は OOM で起動すら不可だった。Phase M の 3 段階 ctx (1024/2048/4096) で起動成功、3 係数分離に成功。これで Qwen3.5-122B-A10B の P100 4 枚構成における **fa=0 の動作可能 ctx 範囲 [1024, 4096] が完全に特定**。
